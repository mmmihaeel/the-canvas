# Security Policy

## Supported Versions

| Version line | Support status |
| --- | --- |
| Latest release | Supported |
| `main` | Supported for active development |
| Older tags | Best effort only |

## Reporting a Vulnerability

Please do not open a public issue for an undisclosed vulnerability.

Report security findings to the repository owner through GitHub and include:

1. Affected area or endpoint.
2. Reproduction steps.
3. Expected impact.
4. Suggested remediation, if available.

If GitHub private vulnerability reporting is enabled for the repository, prefer that channel. Otherwise, contact the maintainer through the profile listed at `https://github.com/mmmihaeel`.

## Response Expectations

- Initial acknowledgement: best effort.
- Triage: confirm severity, affected scope, and mitigation path.
- Remediation: patch on `main`, publish a follow-up release, and document relevant operator actions.
